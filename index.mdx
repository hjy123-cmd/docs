## 平台功能

<CardGroup cols={3}>
  <Card title="日志采集" icon="server">
    Filebeat Agent 部署在所有业务服务器，自动采集本地日志文件，零侵入、低资源占用。
  </Card>
  <Card title="高可用缓冲" icon="layer-group">
    Kafka 集群（KRaft 模式）作为消息中间件，保障日志流量峰值时的可靠传输与削峰填谷。
  </Card>
  <Card title="清洗与脱敏" icon="filter">
    Logstash 提供灵活的 Pipeline 过滤规则，支持字段提取、格式转换和敏感数据脱敏。
  </Card>
  <Card title="弹性存储" icon="database">
    Elasticsearch 集群实现冷热数据分离，ILM 策略自动管理索引生命周期，降低存储成本。
  </Card>
  <Card title="可视化检索" icon="chart-bar">
    Kibana 提供全文检索、字段过滤和可视化大屏，快速定位和分析业务日志。
  </Card>
  <Card title="统一监控" icon="gauge">
    Prometheus + Grafana 监控所有组件健康状态，Alertmanager 支持企业微信、邮件、短信告警。
  </Card>
</CardGroup>

## 快速导航

<Steps>
  <Step title="部署 Filebeat Agent">
    按照[安装指南](/collection/filebeat-install)在业务服务器上安装 Filebeat，并完成基础配置。
  </Step>
  <Step title="配置日志采集">
    编辑 [Filebeat 配置文件](/collection/filebeat-config)，指定日志路径并将数据输出到 Kafka。
  </Step>
  <Step title="配置 Logstash Pipeline">
    设置 [Logstash Pipeline](/processing/logstash-pipeline) 进行日志解析、过滤与写入 Elasticsearch。
  </Step>
  <Step title="在 Kibana 中查询日志">
    登录 [Kibana](/visualization/kibana-overview)，使用 Discover 功能检索和分析您的日志数据。
  </Step>
</Steps>
