本指南帮助您快速完成从业务服务器到日志监控平台的完整接入流程。您将安装 Filebeat Agent，配置日志输出到 Kafka，并在 Kibana 中验证日志已成功采集。整个过程预计需要 5 分钟。

<Note>
  开始之前，请确认您拥有业务服务器的 SSH 访问权限，以及 Kibana 的登录账号。
</Note>

<Steps>
  <Step title="安装 Filebeat">
    在业务服务器上以 root 权限执行以下命令安装 Filebeat：

    <CodeGroup>

    ```bash RPM (RHEL/CentOS)
    # 下载并安装 Filebeat 8.x
    curl -L -O https://artifacts.elastic.co/downloads/beats/filebeat/filebeat-8.13.0-x86_64.rpm
    sudo rpm -vi filebeat-8.13.0-x86_64.rpm
    ```

    ```bash DEB (Ubuntu/Debian)
    # 下载并安装 Filebeat 8.x
    curl -L -O https://artifacts.elastic.co/downloads/beats/filebeat/filebeat-8.13.0-amd64.deb
    sudo dpkg -i filebeat-8.13.0-amd64.deb
    ```

    </CodeGroup>

    安装完成后验证版本：

    ```bash
    filebeat version
    ```
  </Step>

  <Step title="配置 Filebeat 输出到 Kafka">
    编辑 `/etc/filebeat/filebeat.yml`，配置日志输入和 Kafka 输出：

    ```yaml /etc/filebeat/filebeat.yml
    filebeat.inputs:
      - type: log
        enabled: true
        paths:
          - /var/log/your-app/*.log
        fields:
          service: your-service-name
          env: production
        fields_under_root: true
        multiline.pattern: '^[0-9]{4}-[0-9]{2}-[0-9]{2}'
        multiline.negate: true
        multiline.match: after

    output.kafka:
      hosts: ["kafka-broker1:9092", "kafka-broker2:9092", "kafka-broker3:9092"]
      topic: 'logs-%{[service]}'
      codec.json:
        pretty: false
      required_acks: 1
      compression: gzip
    ```

    <Tip>
      将 `your-app` 替换为您的实际应用名称，`your-service-name` 替换为服务标识符，Kafka Broker 地址请咨询平台管理员。
    </Tip>
  </Step>

  <Step title="启动 Filebeat 服务">
    检查配置文件语法并启动服务：

    ```bash
    # 验证配置
    sudo filebeat test config -e

    # 测试 Kafka 连通性
    sudo filebeat test output -e

    # 启动并设置开机自启
    sudo systemctl enable filebeat
    sudo systemctl start filebeat

    # 查看运行状态
    sudo systemctl status filebeat
    ```
  </Step>

  <Step title="在 Kibana 中验证日志">
    登录 Kibana，进入 **Discover** 页面：

    1. 在左上角选择索引模式，例如 `logs-*`
    2. 将时间范围设置为 **最近 15 分钟**
    3. 在搜索框输入您的服务名称，例如：`service: your-service-name`
    4. 确认日志条目出现在列表中

    <Note>
      如果 15 分钟后仍未看到日志，请参阅[排查 Filebeat 采集问题](/ops/troubleshooting)。
    </Note>
  </Step>
</Steps>
